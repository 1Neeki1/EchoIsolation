## Система взаимодействия

Все игровые объекты используют единый интерфейс.

Структура объекта:

* mesh
* name
* action

Игрок взаимодействует через клавишу E.

Примеры:

* двери
* терминалы
* аудиодневники
* генераторы
* лифты

Запрещено создавать отдельную систему взаимодействия для каждого типа объекта.

## Этап разработки Prototype

До версии 0.2 проект остаётся в одном файле:

* game.js

Причина:

Быстрая разработка и тестирование механик.

После появления:

* системы взаимодействия
* аудиодневников
* паники
* эхолокации
* Слепца

выполняется рефакторинг в отдельные модули.

## Система сообщений

Используется существующий элемент:

#message

Все игровые подсказки, цели и взаимодействия выводятся через него.

Примеры:

[E] Прослушать запись

Батарея подобрана

Фонарик разряжен

Найти резервное питание

Запрещено создавать дополнительные элементы уведомлений без необходимости.

## Система Interactable

Все интерактивные объекты регистрируются в массиве:

interactables

Структура:

{
mesh,
name,
action
}

Игрок ищет ближайший объект в радиусе взаимодействия.

После нажатия E вызывается action().
